# Growth Screener — Agent Skill

## 目標定位

這個工具是**研究起點，不是交易信號**。
目標是從 S&P 500 + NASDAQ-100 的 500+ 支中大型股中，每天縮小到 15–25 支值得深入研究的候選標的。

**使用情境：**
> 每天執行一次，找出「近期技術面強勢突破、基期夠低、基本面不壞」的股票，搭配新聞自動解釋原因，節省大海撈針的時間。

## 核心篩選邏輯（signals.py）

### 四個硬性條件（全部需通過）

| 條件 | 參數 | 設計原因 |
|------|------|----------|
| 5日累積漲幅百分位 | ≥ 85% | 相對自己的歷史，今天這波漲幅是否異常強勢（避免用絕對值，每支股票波動性不同） |
| 連續上漲天數 | 5天內 ≥ 3天正報酬 | 確認方向持續，排除單日急漲後立即回落 |
| 基期夠低 | 10天前股價距 6個月高點 ≤ -15% | 確認是從低位突破，而非追高；排除已在高位繼續漲的情況 |
| 均線突破 | 收盤 > MA20 | 確認已重新站上短期均線，突破有效 |

### 評分公式（決定排序）

```
Score = 5D百分位 × 0.35
      + 基期深度得分 × 0.25     # 越深的基期 → 越高的回彈潛力
      + 連續天數得分 × 0.20     # consec_up / 5
      + 均線品質得分 × 0.20     # MA20上升 + 站上MA50 + 黃金排列
```

### 均線品質三項指標
- **MA20 上升**：現在 MA20 > 10天前的 MA20（趨勢向上）
- **站上 MA50**：確認中期趨勢轉正
- **黃金排列（MA20 > MA50）**：短中期趨勢對齊

## 基本面過濾邏輯（info.py）

### 死貓跳 vs. 真實轉折的判斷

這是最關鍵的設計決策：**技術面強勢不代表值得研究，需要排除「跌夠多短暫反彈、業務仍在惡化」的情況**。

**三層分類：**

```
基本面 OK
  → ✅ 正常列出

基本面有問題 + 股價已回到 52 週高點附近
  → ⚠ 保留並標記警告
  → 原因：市場已定價轉型預期，可能有題材，值得自行驗證
  → 典型案例：Intel (INTC) —— FCF -$4.5B 但股價創近期新高

基本面有問題 + 股價仍在低位（未回到高點）
  → ✗ 移除
  → 原因：市場尚未認可反彈，死貓跳機率高
```

**死貓跳判斷條件（任一符合即觸發）：**
- 營收年增率 < -10%（業務結構性衰退）
- 營收下滑 AND 自由現金流 < -$5億（衰退且燒錢）
- 分析師平均評等 ≥ 4.0（Sell / Strong Sell 共識）

**重要設計原則：** 若 yfinance 拿不到基本面資料，預設通過（不因資料缺失而誤殺）。

## 顯示欄位說明

| 欄位 | 說明 |
|------|------|
| ★ | 股價在 52 週高點 5% 以內（突破型） |
| ⚠ | 基本面有疑慮但技術面已復原，需驗證轉型題材 |
| ↑/5d | 過去 5 個交易日中幾天為正報酬，例如 5/5 = 全部上漲 |
| Base | 這波漲勢開始前（10天前），股價距 6 個月高點的距離 |
| MA | ↑ = MA20 上升；50 = 站上 MA50；✦ = 黃金排列 |
| RevGr | 營收年增率（綠 ≥ +10%、黃 0–10%、紅 < 0%） |
| FCF | 自由現金流（正值=健康，負值=燒錢）；後附 EPS 方向箭頭 |
| Rating | 分析師綜合評等 |
| 5D Rank | 5日漲幅在自己過去 252 天中的百分位 |

## 回測方法論

### 設計原則
- **無 look-ahead bias**：每個歷史篩選日只使用當天及之前的資料
- **固定持有期**：信號日 + 63 個交易日（≈ 3 個月）
- **Benchmark**：同期 SPY 報酬

### 執行方式
- 下載 2 年歷史資料（需要 1 年做指標計算 + 1 年做回測）
- 每 5 個交易日執行一次篩選（≈ 每週）
- 統計信號的勝率、平均報酬、超額報酬分佈

### 已知限制
- **基本面過濾無法回測**：yfinance 只提供當前基本面，無法取得歷史財報時間點資料（point-in-time）。因此 backtest.py 的數字不包含基本面過濾的效果。
- **回測週期偏短**：目前約 9 個月有效回測期，樣本數 ~900 個信號，結論方向正確但不宜過度解讀。
- **未來事件盲點**：部分信號發生在財報前，財報結果可能使 3 個月報酬大幅偏移。

## 策略演進與學習

### v1：單日 RVOL 篩選
- 條件：單日漲幅百分位 ≥ 90% **且** RVOL ≥ 2x
- 問題：大盤普漲日（如 2026/4/14）成交量分散，MSFT/META/MU 等明顯強勢股都被過濾掉
- 回測：勝率 61.7%、平均 3M +8.1%、alpha +2.6%

### v2：加入基期低
- 條件：移除 RVOL 硬篩選，改用「基期 ≤ -15%」
- 學習：RVOL 適合找有特定催化劑的個股（如 ORCL 的 Bloom Energy 合約），不適合大盤普漲日
- 回測：勝率 66.2%、平均 3M +9.6%、alpha +0.6%

### v3：多日累積 + 連續天數 + 均線（現行版本）
- 條件：5 日百分位 + consec ≥ 3 + 基期低 + price > MA20
- 學習：
  - 5 日比單日更能過濾「一日暴衝後立即回落」的假訊號
  - 連續上漲確認方向持續性
  - MA20 突破作為硬條件確保有效性
- 回測：勝率 68.1%、平均 3M +9.8%、alpha +1.2%

### 基本面過濾層（研究用）
- 設計動機：區分「真實轉折」與「死貓跳」
- 關鍵洞察：**價格是否回到 52 週高點** 是最有力的區分指標
  - 若已回到高點 → 市場已定價轉型 → 保留並標記 ⚠
  - 若仍在低位 → 反彈尚未被市場認可 → 移除

## 未來可改進的方向

1. **盈餘動能**：加入「分析師 EPS 預估修正」篩選（需付費資料或 earnings whispers）
2. **財報日過濾**：標記未來兩週內有財報的股票，提醒「買預期賣結果」風險
3. **產業強度**：同類股是否整體在上漲（減少個股判斷噪音）
4. **市場環境判斷**：SPY > SPY_MA50 才信任信號（在熊市中信號失效率高）
5. **5 日累積成交量**：用 5 日總量比較均量，比單日 RVOL 更穩定
